Adequacy of PCF

This was the main project of my masters degree, for which I wrote a report. After formalising a simple imperative lamguage in Agda for my last mini project, I wanted to work on a more complicated language, and look at recursion in particular.

I chose PCF for this reason. Programming Computable Functions (PCF) is a simple function programming language and is an extension of the Simply Typed Lambda Calculus with recursion defined by fixpoint recursion.

In my project I defined the operational and denotational semantics of PCF and proved the Soundness of the operational semantics with telation to the denotational semantics. I also proved a property called Type Safety. In the previous mini project I proved Completeness of the two approaches to the Sematics of my language. In PCF, we cannot prove this theorem, but instead prove a simpler property called Adequacy.

This property states that any closed term of base type (natural number type) evaluates to the same expression in both version of the semantics and that this can be proved in both directions. To prove this, I studied a new proof technique called Logical Relations. I also discussed how the Logical Relations technique can be used to prove that there are some elements of the denotational model that do not correspond to any programs we can define in PCF, as described in Thomas Steicher's textbook.
